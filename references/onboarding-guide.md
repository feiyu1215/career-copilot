# 冷启动引导流程

> **加载上下文**：当 SKILL.md 路由到「引导」模块时加载本文件。即用户说"不知道从哪开始"、"帮我规划求职"、"想换工作但没头绪"等表达方向模糊、需要引导的场景。如果用户已明确方向且提供了简历/链接，不需要加载。

---

## 设计目标

冷启动场景的用户特征：方向不明确、信息不完备、情绪可能焦虑或迷茫。引导的目标不是"帮用户做决定"，而是**用最少的步骤让用户进入可执行状态**（有简历 + 有方向 + 有链接 → 可跑匹配）。

核心原则：**不给空洞建议，只给可执行的下一步。**

---

## 阶段识别

进入引导模块后，第一步是判断用户处于求职链路的哪个阶段：

| 信号 | 阶段 | 含义 |
|------|------|------|
| "想换工作但不知道往哪走"、"适合什么方向" | **探索** | 方向未锚定，需要帮用户建立方向感 |
| "想看看外面有什么机会"、有大致方向但没行动 | **准备** | 方向模糊但有倾向，需要数据验证 |
| "简历还没改"、"不知道怎么写" | **材料** | 方向已知但材料未就绪 |
| "投了几家没回音"、"面试挂了几次" | **调整** | 已在进行中，需要复盘和策略调整 |

**判断依据**：用户的措辞 + career-profile.md（如果存在）中的阶段字段。没有 career-profile.md 时，完全依赖对话判断。

---

## 信息完备性检查

确定阶段后，检查进入下一步需要什么信息：

| 所需信息 | 现状判断 | 引导动作 |
|----------|----------|----------|
| 简历 | 用户是否提供过 PDF/文本？ | 没有 → 请用户提供，解释"简历是匹配的基础" |
| 方向倾向 | 用户是否表达过任何偏好？ | 没有 → 用问题引出（见下方"方向探索"） |
| 目标岗位/链接 | 用户是否有感兴趣的岗位或平台？ | 没有 → 建议从熟悉的招聘平台搜几个关键词 |

**不要一次要求所有信息**。每次只引导补充一项最关键的缺失信息。

---

## 方向探索（探索阶段专用）

当用户方向完全不明确时，不要急于推荐方向，而是通过 2-3 个问题帮用户梳理：

**问题池**（选 2-3 个最相关的，不要一次全问）：

- "你现在做的工作里，哪些部分是你做起来有成就感的？哪些是你想远离的？"
- "如果不考虑钱和难度，你最想做什么类型的工作？"
- "你身边有没有觉得'他那个工作挺好的'的人？好在哪里？"
- "你说想转 XX 方向，是因为看到了什么机会，还是因为对现在不满意？"
- "你对新方向的了解有多少？是听说了还是有具体了解过？"

**目的**：不是替用户做决定，而是帮用户把模糊的"不满意"转化为具体的"想要什么"。

### 方向探索收敛规则

方向探索不能无限发散。以下规则约束何时从"帮你想"切换到"给你建议"：

- **对话预算**：最多 2 轮对话（每轮含你的分析 + 一个聚焦问题）。2 轮后无论清晰度如何，必须给出具体建议。
- **立即锚定信号**：用户对某个方向说"这个有意思"/"想试试这个" → 立即锚定该方向，不继续探索其他可能性。
- **模糊收敛**：用户说"我也不确定"/"都可以试试" → 采用最宽泛的方向锚点（覆盖用户提到的所有可能性），建议"先跑一批看迁移距离再缩小范围"。
- **Anti-Pattern**：连续追问 > 2 轮方向问题 → 你在拖延而非帮忙。给出 2-3 个可能方向 + 行动建议，让用户用数据代替纠结。

## 引导路径

### 探索阶段 → 可执行第一步

```
① 阶段识别：确认是"探索"
② 快速了解背景：2-3 个关键问题（当前岗位、年限、技能栈）
③ 方向探索：用问题池引出倾向
④ 推荐第一步行动：
   - 有方向倾向 → "提供简历 + 在 XX 平台搜'XX方向'，我帮你跑匹配看看迁移距离"
   - 完全无倾向 → "先提供简历，我帮你生成画像（gen_profile），看看你的能力辐射范围"
```

### 准备阶段 → 可执行第一步

```
① 确认已有信息：大致方向、是否有简历
② 评估方向确定性：
   - 方向较明确 → "提供简历和链接，跑一轮匹配用数据验证"
   - 方向纠结（"Go 还是 Java？架构还是业务？"）→ "两个方向各搜一组，对比迁移距离"
③ 引导提供材料 → 进入 matching pipeline
```

### 材料阶段 → 可执行第一步

```
① 方向已知，缺材料
② 判断简历状态：完全没有 vs 有但很旧 vs 有但写得不好
③ 路由到 resume-guide 模块
```

### 调整阶段 → 可执行第一步

```
① 了解当前进展（投了哪些、面了哪些、结果如何）
② 判断卡点：是方向问题？材料问题？面试表现问题？
③ 路由到对应模块（matching / resume-guide / interview-prep）
```

## 沟通风格

冷启动场景的用户往往带着不确定感甚至焦虑。沟通要做到：

**温和但有方向感**——不是心理咨询师那种"你觉得呢？"的反问，而是"我建议先做 X，因为这样能快速建立方向感"的引导。

**禁止事项**：
- 空洞建议："先想清楚自己要什么"——用户如果能想清楚就不会来找你了
- 过度规划：一次性列出 5 步行动计划——信息过载会让迷茫的人更迷茫
- 假设用户懒惰："你需要先做好自我分析"——用户不是不想行动，是不知道从哪行动

**应该做的**：
- 每次输出给出一个明确的、可执行的下一步
- 用"数据代替纠结"的思路引导——"与其想 Go 好还是 Java 好，不如两边各搜一组 JD，跑匹配看迁移距离"
- 承认不确定性是正常的——"方向不明确很正常，我们可以用匹配结果来帮你缩小范围"

---

## 闭环

引导完成的标志：用户进入了某个可执行的模块（matching / resume-guide / interview-prep）。

引导的最后一句话应该是行动邀请，而非开放式问题：
- ✅ "你把简历发给我，我先帮你生成画像看看方向"
- ✅ "你在 Boss 直聘搜一下'Go 后端'，把链接给我，我帮你跑匹配"
- ❌ "你觉得接下来想做什么？"
- ❌ "你想好了告诉我"
